# Data Analysis Portfolio

## 概要
本リポジトリは、**実務を想定したデータ分析ポートフォリオ**です。  
分析設計からデータ理解、判断・施策提案までの一連の流れを、  
3つのケーススタディとして整理しています。

本ポートフォリオでは特に、以下を重視しています。

- 課題に応じた **分析手法・技術の使い分け**
- SQL / Python / 統計を役割分担して用いる分析設計
- 分析結果をもとにした判断・ネクストアクションの提示
- 再現性を意識した分析環境の構築

---

## ケース概要（技術の使い分け）

### Case1：SQLによる分析設計と施策テーマ設計
**使用技術**：SQL（BigQuery）

レーティングデータを起点に、  
集計・結合・共起分析などをSQLで完結させ、  
マーケティング施策として成立する特集テーマを設計するケースです。

📂 `case01/`

---

### Case2：SQL × Pythonによる指標設計とEDA
**使用技術**：SQL / Python

SQLで顧客単位の指標・特徴量を設計した上で、  
Pythonによる探索的データ分析（EDA）を行い、  
施策設計やモデル化を行う前提条件が成立するかを検証するケースです。

📂 `case02/`

---

### Case3：Pythonによる前処理・モデリング・統計検定
**使用技術**：Python / 統計的検定

Pythonで前処理・特徴量設計を行い、  
ロジスティック回帰による反応予測と統計検定を組み合わせ、  
施策判断につなげるケースです。

📂 `case03/`

---

## 技術の使い分け方針
- **SQL**：データ抽出・集計・特徴量設計  
- **Python**：EDA・可視化・前処理・モデリング  
- **統計手法**：仮説検証・結果の妥当性確認  

分析手法ありきではなく、  
**課題と判断内容に応じて最適な手段を選択する**ことを意識しています。

---

## 分析環境（再現性）
本ポートフォリオでは、分析結果の再現性を担保するため、  
**pyproject.toml による依存管理**と  
**Docker / docker-compose による実行環境の固定**を行っています。

- `pyproject.toml`：Python依存関係の管理  
- `docker-compose.yml`：分析環境の起動設定  
- `docker/`：Dockerfile 等の環境定義  

### 技術の使い分け方針

**課題・判断内容に応じて最適な手段を選択**することを意識しています。
